# ShoeCloud | 云鞋库 — 智能跑鞋管理平台

## 🎯 项目概述

**ShoeCloud（云鞋库）** 是一款专为严肃跑者设计的智能跑鞋管理应用。它通过创新的技术方案，解决了跑者手动记录跑鞋里程繁琐、装备数据分散的核心痛点，并基于真实使用数据构建了一个无广告、可信赖的跑鞋评价生态系统。

> **核心理念**：连接运动数据与装备实体，让每一次奔跑都被精准记录，让每一双跑鞋的价值都被充分发掘。

---

## ✨ 核心功能架构

### 1. 🚀 智能自动化同步（产品基石）

- **“碰一碰”极速关联**：通过 NFC 芯片或软件模拟，跑步后手机轻触跑鞋标签，自动完成“本次跑步↔这双鞋”的绑定
- **全平台数据接入**：已正式接入 **Garmin Connect API** 与 **COROS Open API**（覆盖超70%核心跑者），数据同步安全合规
- **零手动记录**：彻底解放用户，跑鞋里程、磨损状态全自动计算更新

### 2. 📊 深度个人装备管理

- **智能寿命预测引擎**：基于跑步里程、用户体重、路面类型等多维度数据的算法模型，提供科学的更换建议
- **完整电子档案**：为每双跑鞋建立从购入到退役的全生命周期数字履历
- **可视化数据洞察**：
  - 📈 实时里程追踪与磨损进度
  - 💰 单次跑步成本与总投入分析
  - 📋 可分享的精美跑鞋报告卡片
- **跑鞋日记功能**：记录与每双战靴共同经历的特殊时刻与赛道回忆

### 3. 🌱 真实可信的社区生态（核心特色）

- **“里程验证”评价机制**：仅当用户实际使用某双跑鞋达到一定里程（如50km+）后，方可对其关键指标进行评价

- **结构化多维评价体系**：

  ```
  中底反馈 · 鞋面包裹 · 大底耐磨 · 稳定支撑 · 透气性
  ```

- **精准标签筛选**：评价可按 **跑者体重**、**足弓类型**、**配速区间**、**使用场景** 等多维度精细化筛选

- **纯净无广告环境**：严格禁止营销内容，打造纯粹为跑者服务的决策平台

- **社群扩展**：基于地理位置的同城跑友发现、装备趋势观察与线下活动组织

## 🏗️ 技术架构与选型

### 前端应用层

```yaml
框架: Flutter 3.38 + Dart 3.10
架构: MVVM with Provider (状态管理)
目标平台: iOS & Android 原生级体验
核心依赖:
  - dio: 网络请求与API管理
  - nfc_manager: NFC功能支持
  - provider: 应用状态管理
  - syncfusion_flutter_charts: 数据可视化
  - hive: 本地轻量数据存储
```

### 后端服务层

```yaml
语言: Python 3.10+
框架: FastAPI (高性能异步API框架)
数据库: 
  - PostgreSQL (主业务数据)
  - Redis (缓存与会话管理)
核心服务:
  1. 数据同步服务: 处理Garmin/Coros API拉取与数据清洗
  2. 用户服务: 账户、权限与个人数据管理
  3. 算法服务: 跑鞋磨损预测模型
  4. 社区服务: 评价数据存储与检索
部署: 容器化部署于云平台，支持弹性扩缩容
```

### 第三方集成

- ✅ **Garmin Connect API** (已申请接入)
- ✅ **COROS Open API** (已申请接入)
- 🔄 **华为运动健康** (规划中)
- 🔄 **Apple HealthKit** (iOS生态集成)
- 🔄 **Strava API** (社区功能拓展)

## 📄 项目合规与隐私

### 数据安全承诺

1. **最小化收集**：仅收集实现功能所必需的数据
2. **端到端加密**：所有数据传输均通过HTTPS加密
3. **用户完全控制**：用户可随时导出个人数据或永久删除账户
4. **透明政策**：完整的隐私政策公开可用，明确说明数据使用方式

### 第三方数据使用

- 严格遵循Garmin、COROS等平台的API使用条款
- 绝不存储用户在第三方平台的登录凭证
- 用户可随时在应用中撤销对任何平台的授权
